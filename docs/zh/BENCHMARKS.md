# Benchmark

AgentLedger benchmark 是发布质量门禁，不是营销性能数字。它同时做两件事：确认 runtime 语义面仍然完整覆盖，并记录本机关键路径耗时，方便后续回归对比。

默认 benchmark 的 runtime 场景不依赖外部服务：

- SQLite `StateStore`
- `LocalBlobStore`
- 不调用真实模型 provider
- 不连接真实数据库、对象存储、Docker daemon 或云服务
- 可选执行 Python、Go、TypeScript、Rust 的本地 conformance CLI 计时

## 运行

在仓库根目录执行：

```bash
PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src python3 scripts/benchmark_runtime.py --iterations 20 --output-dir /tmp/agentledger-benchmark
```

开发时快速 smoke：

```bash
PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src python3 scripts/benchmark_runtime.py --iterations 1 --skip-language-commands --output-dir /tmp/agentledger-benchmark-smoke
```

输出内容：

- `benchmark.json`：机器可读结果，包含 samples、coverage matrix、validation failures 和命令输出摘要。
- `benchmark.md`：给 release review 看的可读报告。
- `run-*/`：每次运行隔离出的 runtime、evidence、静态 HTML 等中间产物。

## 覆盖范围

benchmark 会读取 `contracts/conformance/runtime_semantics.v1.json`，并在 `coverage_matrix` 中列出每个 required semantic check。

当前覆盖包括：

- durable run、persistence、replay、evidence export
- local blob store
- tool schema validation
- worker/service 和 scheduler 行为
- Tool Ledger 幂等重试
- policy、approval、sandbox fail-closed
- cost 和 failure attribution
- media artifact 与 stream checkpoint evidence
- evidence consumers、trace、OTLP JSON、Inspector HTML、time-travel HTML
- ops readiness、storage schema helper、非破坏性的 retention/backup 检查
- MCP 和 framework adapter contract
- boundary lint
- adversarial review 和 evidence regression
- failure injection
- shadow/repro harness primitives
- optional / official adapter 的 certification 与 dry-run 边界

如果开启语言命令，Go、TypeScript、Rust conformance CLI 必须报告共享 manifest id。完整成功时应看到：

```text
required_check_count = 27
covered_check_count = 27
not_run_count = 0
by_status.measured_and_language_conformance = 27
```

## 如何理解结果

耗时数字只用于同一机器上的回归对比，不代表跨机器、跨 CI、跨操作系统或跨 Python 版本的通用性能结论。

benchmark 会刻意覆盖非 happy path：

- side effect 成功后 crash，再安全 retry
- invalid tool input 在执行前被拒绝
- approval pause/resume
- required sandbox fail-closed
- failure injection：retry exhaustion、stale lease fencing、cancellation fencing、side-effect idempotency
- boundary lint 捕获一个故意写出的 direct shell call

部分 adapter 相关检查是 contract/dry-run，不是生产环境证明。Postgres、MySQL、S3、Docker、Langfuse、OTLP collector、Temporal 等仍需要单独的真实服务验证，才能做 production claim。

## 发布使用

runtime-core 或多语言 release 前建议执行：

```bash
PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src python3 scripts/benchmark_runtime.py --iterations 20 --output-dir /tmp/agentledger-benchmark
python3.11 scripts/check_language_parity.py
```

如果 release 改了 runtime 语义、failure handling、Inspector/debug output、evidence consumers、adapter 或 language parity，建议在 release notes 中摘要 `benchmark.md`。
