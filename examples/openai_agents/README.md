# OpenAI Agents SDK-style Approval And Replay

This dependency-free example shows how an OpenAI Agents SDK-style runner can keep AgentLedger's runtime boundary around model evidence, proposed tool calls, approval, Tool Ledger, evidence export, and replay.

It does not import the real OpenAI Agents SDK and does not call the network. The local facade models the integration shape:

```text
SDK-style agent run
  -> record model request/response evidence
  -> record model-proposed tool call
  -> AgentLedger ToolGateway
  -> approval / Tool Ledger / evidence / replay
  -> external tool handler
```

Run from the repository root:

```bash
PYTHONPATH=src python3 examples/openai_agents/approval_replay.py
```

Expected behavior:

- the fake SDK-style runner records model request/response evidence
- the model-proposed `ticket.create` call is recorded before execution
- the first runtime attempt pauses for approval
- after approval, the second attempt reuses the archived proposal and executes the external-write tool once
- the output includes the run id, Tool Ledger row, approval record, replay summary, evidence directory, and Inspector HTML path

This is an ecosystem example, not OpenAI endorsement, certification, or an official SDK binding. A production integration should replace the local facade with the real SDK object and keep model/tool side effects inside the AgentLedger runtime boundary.
