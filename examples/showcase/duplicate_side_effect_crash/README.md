# Showcase: Crash After Tool Success

This is the short, shareable demo for AgentLedger.

It tells one story:

```text
An agent sends an email.
The email succeeds.
The worker crashes before checkpoint.
The task is retried.
```

Without a runtime ledger, the retry sends the email again. With AgentLedger, the Tool Ledger and idempotency key let the retry reuse the recorded side effect instead of duplicating it.

Run from the repository root:

```bash
PYTHONPATH=src python3 examples/showcase/duplicate_side_effect_crash/demo.py
```

Expected output:

- `WITHOUT runtime ledger` shows `external_email_count_after_retry: 2`
- `WITH AgentLedger` shows `external_email_count_after_retry: 1`
- `actual_tool_executions: 1`
- `replay_safe: true`
- generated `runs.html`
- generated `inspector.html`

Open the generated HTML files to see:

- run status and event timeline
- Tool Ledger row with `SUCCEEDED`
- failure/retry evidence
- replay-safe evidence
- full JSON details for audit/debug

To choose the output directory:

```bash
AGENTLEDGER_SHOWCASE_ROOT=/tmp/agentledger-showcase PYTHONPATH=src python3 examples/showcase/duplicate_side_effect_crash/demo.py
```

This demo is intentionally small. It is meant for screenshots, GIFs, README walkthroughs, and quick project evaluation.
