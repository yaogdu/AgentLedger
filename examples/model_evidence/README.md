# Model Evidence Boundary

This example shows how to use AgentLedger when your app, framework, provider SDK, or model gateway already executed the model call.

AgentLedger does not route the model request here. It records what happened so later debugging can answer:

- what request was sent to the model
- what raw response came back
- whether the model proposed a tool call
- whether a model failure was timeout, rate limit, malformed output, or provider error
- how the model evidence connects to ToolGateway execution, failure lifecycle, Inspector, and exports

Run from the repository root:

```bash
PYTHONPATH=src python3 examples/model_evidence/demo.py
```

The output includes Inspector read-model counts for `model_calls` and `tool_proposals`.
